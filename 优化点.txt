6、流式输出

stream_mode="messages"：
这是 LangGraph 最新的特性。以前我们只能流式传输状态（State），现在可以直接流式传输消息块（Message Chunk）。
当 LLM 生成一个字，LangGraph 就 yield 一个 (message_chunk, metadata) 元组。

curr_node 过滤：
在你的图中，summarize_node 也会调用 LLM（千问 Turbo）来写摘要。
如果不加 if curr_node == "summarize_node": continue，用户会在屏幕上突然看到一段莫名其妙的“摘要文本”混在回答里。我们通过元数据把它过滤掉，让它在后台静默运行。

yield full_response：
Gradio 的流式要求函数是一个 Python Generator。
每次 yield，聊天框的内容就会刷新一次，从而产生打字机的效果。

高级功能：

1. 深度人岗匹配分析 (可视化能力) 📊

痛点：目前只返回了“推荐职位列表”，用户不知道为什么匹配，也不知道自己缺什么。 功能：
岗位匹配度雷达图：
让 LLM 对比“用户简历”和“目标职位 JD”，从 5 个维度打分（如：技术栈、工作年限、学历、项目经验、软技能）。
使用 matplotlib 或 Gradio 的 gr.Plot 组件，在聊天框中直接渲染一个雷达图。
直观效果：用户一眼就能看出“这个岗位我技术够了，但年限差点”。
Gap 分析与提升建议：
不仅推荐职位，还指出“为了通过面试，你还需要补充 XXX 知识”。


2. AI 模拟面试官 (Agent 交互能力) 🎙️

痛点：找完工作后，用户最大的焦虑是面试。 功能：
情景模拟：
用户选中一个推荐的职位，点击“开始模拟面试”。
Agent 切换角色（System Prompt 变更），扮演该公司的面试官。
它会根据 JD 里的要求，针对用户的简历挖坑提问（例如：“我看你简历里写精通 Redis，能讲讲你项目中遇到的缓存穿透是怎么解决的吗？”）。
面试复盘：
模拟结束后，Agent 给出评分和改进建议。


3. 定制化文书生成 (生成能力) 📝

痛点：投递简历时，针对不同公司写 Cover Letter（求职信）很累。 功能：
一键生成求职信：
用户选择一个职位，点击“生成求职信”。
Agent 结合 用户简历的优势 和 该职位 JD 的痛点，自动生成一篇高情商的求职信/打招呼语（用于 Boss 直聘）。
简历针对性优化：
针对特定大厂岗位，自动重新改写简历中的“项目描述”，使其更契合该岗位的关键词（Keyword Stuffing）。


4. 数据看板与看板管理 (业务流能力) 📉

痛点：用户可能投了多个岗位，记不住状态；或者想了解市场行情。 功能：
薪资趋势图：
利用 analyze_trend_tool 的数据，画出折线图或柱状图，展示该岗位的薪资分布（如：北京 Java 平均 25k，中位数 22k）。
投递看板 (Kanban)：
在左侧栏增加一个“投递记录” Tab。
用户可以把感兴趣的职位“收藏”，并标记状态（已投递、约面试、挂了、Offer）。
