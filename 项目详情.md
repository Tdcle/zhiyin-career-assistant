# 职小助——智能求职 AI Agent (RAG) 项目技术路线与架构梳理

## 一、 项目概述

本项目是一个基于大语言模型（LLM）和检索增强生成（RAG）技术构建的**一站式智能求职与面试 Agent 生态系统**。

系统不仅提供基于自然语言的高精度岗位检索，还打通了求职的完整闭环：从底层的 **RPA 自动职位抓取与向量化**，到**多模态简历解析 (OCR)**，再到**多 Agent 协同的求职咨询**，最后延伸至**沉浸式 AI 模拟面试与匹配度雷达评估**。系统具备长短时记忆、自动化评测闭环，达到了工业级落地标准。

## 二、 核心技术栈

- **大语言模型 (LLM)**：通义千问系列
  - `qwen-max`：主控 Agent 复杂推理、意图提取、严厉面试官模拟、RAGAS 评测裁判。
  - `qwen-plus`：生成最终深度的 Markdown 面试评估报告。
  - `qwen-turbo`：高频次的历史会话记忆压缩 (Summarize)、职位摘要生成。
- **向量模型 (Embedding)**：本地 Ollama (`nomic-embed-text-v2-moe`)，将长文本转为高维稠密向量。
- **Agent 编排框架**：LangGraph (构建复杂的图结构多智能体工作流，实现状态流转与双向通信)。
- **数据采集与处理**：DrissionPage (底层无头浏览器 RPA 与数据包监听)、PaddleOCR + PyMuPDF (多模态文档解析)。
- **数据库与检索引擎**：PostgreSQL + `pgvector` (向量检索) + `pg_trgm` (模糊与正则加速)。
- **UI 与可视化**：Gradio (交互界面)、Matplotlib (人岗匹配雷达图)。
- **评测框架 (Evaluation)**：RAGAS (基于 LLM-as-a-Judge)。

## 三、 系统核心模块与实现细节

### 3.1 极速爬虫与数据工厂引擎

摒弃了传统的静态 DOM 解析，采用更高级的 RPA 抓包方案：

- **数据包监听爬虫**：使用 DrissionPage 直接监听底层的 JSON 数据包 (`wapi/zpgeek/search/joblist.json`)，配合智能平滑滚动模拟真人行为，极大提升了抓取稳定性和速度。
- **异步数据工厂**：通过 `qwen-turbo` 将几千字的冗长职位详情提炼为 100 字以内的核心摘要 (`summary`)，随后调用 Ollama 生成向量嵌入 (`embedding`)，存入 PostgreSQL 供 RAG 系统使用。

### 3.2 多模态简历智能解析系统 (`utils/file_parser.py`)

打造了兼容纯文本与扫描件的智能解析模块：

- **智能降级回退机制 (Fallback)**：优先使用 PyMuPDF (`fitz`) 提取 PDF 文本。如果提取出的有效字符极少（<100字），系统会判定其为扫描件或图片简历。
- **图像张量转换与 OCR**：一旦触发回退机制，系统通过 `get_pixmap(matrix=fitz.Matrix(2, 2))` 将 PDF 页面高清渲染为图像，使用底层数组转换对接 PaddleOCR，精准提取扫描件中的文字，实现图片/PDF简历的无缝解析入库。

### 3.3 工业级多路混合检索架构 (Hybrid Search)

抛弃了传统的“纯向量检索”盲盒，采用了一套精妙的 SQL 级混合召回+精排策略：

- **意图结构化提取 (Tool Calling)**：Agent 将自然语言转化为 `city`, `experience`, `semantic_query`。
- **第一层：硬条件前置过滤 (Hard Filter)**：在 SQL 的 `WHERE` 子句中使用 `ILIKE` 精确截断城市/经验。
- **第二层：向量语义召回 (Dense Retrieval)**：使用 `<=>` 算子计算 `semantic_query` 向量距离，选取 Top N。
- **第三层：正则提分精排 (Regex Scoring/Sparse)**：将用户 Query 转为正则表达式 (`~*`)。若 `title` 或 `summary` 直接命中关键词，则强制缩小向量距离（如 `vec_dist * 0.85`），将精准结果排名置顶。

### 3.4 职小助主大脑 Agent 工作流 (`models/chat_graph.py`)

使用 LangGraph 构建的图结构智能体：

- **多工具路由**：根据用户意图，动态调用 `search_jobs_tool`、`analyze_trend_tool`、`save_preference_tool` 等。
- **UI/大模型解耦设计 (Extract Node)**：创新性地设计了 `extract_search_results` 节点。当工具执行完毕后，在流转回 LLM 之前，先拦截并提取结构化的 UI 卡片数据（写入 Graph State），让前端 Gradio 能够独立渲染精美的职位卡片按钮，而无需 LLM 费力拼接 Markdown 表格。
- **记忆压缩防爆机制**：基于 `summarize_node` 监控对话深度，当消息超限时，自动调用小模型对历史上下文进行无损压缩，保护 LLM Context Window 不被撑爆。

### 3.5 AI 沉浸式模拟面试系统 (`models/interview_graph.py`)

这是一个独立于求职对话的“第二智能体 (Sub-Agent)”：

- **多维雷达图匹配 (`utils/plotter.py`)**：在面试前，调用 Matplotlib 基于用户简历和岗位 JD 生成五维能力匹配雷达图。
- **双 LLM 节点架构**：
  - **严厉面试官节点 (`interviewer_node`)**：配置为 `qwen-max`，设定为严格的判卷式考核者。具备“边问边记”能力（如果在面试中候选人暴露出新的技能，面试官会自动调用 `save_preference_tool` 更新用户的长期画像）。
  - **总结报告节点 (`report_node`)**：面试结束后控制权转移至此节点，使用非流式的 `qwen-plus`，一次性纵览全局上下文，输出包含“综合评分、亮点、不足、针对性建议、通过概率”的万字级深度 Markdown 评估报告。

## 四、 工业级自动化评测体系 (`eval/`)

为解决“冷启动无标注数据集”的痛点，构建了完全独立且可复用的数据驱动评测流水线。

### 4.1 逆向生成黄金数据集 (`generate_dataset.py`)

- **原理**：利用大模型从数据库已有的职位数据中逆向发问（Synthetic Data Generation）。
- **双意图隔离**：在 Prompt 中严格约束大模型，生成两种完全不同分布的测试用例：
  - **泛化搜索 (Broad Search)**：用户提问不带公司名，仅描述品类需求。旨在测试底层检索引擎的召回率。
  - **精确匹配 (Precise Match)**：用户提问带公司名及具体自身条件。旨在测试全链路 RAG（含大模型生成）的精准度。
- **端到端参数模拟**：大模型不仅生成 `user_question` 和 `reference_answer`，还模拟 Agent 提前提取出了 `semantic_query`、`city` 等入参，彻底消除了评测时的分布偏移陷阱。

### 4.2 双链路自动化跑分 (`evaluate_ragas.py`)

- **纯检索能力评估 (Broad Search)**：
  - 直接统计 `Recall@5` 和 `Recall@10`（实际测试 `Recall@10` 达到 100%）。验证了硬过滤+向量+正则提分架构的极致可靠性。
- **RAGAS 框架生成质量评估 (Precise Match)**：
  - 将检索片段与大模型生成的回答组装为 HuggingFace Dataset，输入 RAGAS 框架。
  - **Context Precision (上下文精确度)**：评估检索排序质量（实际达到 97%+）。
  - **Context Recall (上下文召回率)**：评估关键信息是否被检索到。
  - **Answer Relevancy (回答相关性)**：评估 AI 回答是否直接解决用户问题。
  - **Faithfulness (忠实度/抗幻觉)**：评估大模型是否严格基于已知文档作答，不瞎编薪资。
- **评测报告落盘**：将高层次成绩单导出为 JSON，将每条用例的雷达维度打分导出为 CSV (`utf-8-sig`)，支持历史迭代对比分析，并加入了高并发限流保护 (`RunConfig`) 防止 API Timeout。

## 五、 项目核心技术亮点（面试核心护城河）

1. **多智能体 (Multi-Agent) 协同与状态隔离**：通过 LangGraph 构建了职责分明的“求职辅导”与“模拟面试”双图谱。通过 StateGraph 的精准控制，实现了状态共享、节点拦截（如 `extract_search_results`）和自动化长期记忆注入。
2. **解决“信息孤岛”的多模态数据接入**：从底层的 DrissionPage 监听抓包，到引入 PaddleOCR 解决扫描版 PDF 简历的痛点，彻底打通了真实世界复杂数据的输入链路，大幅降低了用户使用门槛。
3. **低成本、高精度的混合检索架构**：没有盲目引入重型组件，依靠 PostgreSQL (`pgvector` + `pg_trgm`) 的组合拳，将结构化条件过滤、语义泛化、实体词匹配无缝融合于一条高效 SQL 中。
4. **全链路的工业级量化评测体系**：实现了无标注数据下的 RAG 评测飞轮，系统效果可度量、可归因，完全契合目前一线互联网大厂的算法工程标准。
